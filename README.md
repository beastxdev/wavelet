# Wavelet

Wavelet is a personal music streaming application built with Flutter, Node.js, and Lavalink. The Flutter client provides the music experience, the Node.js backend stores local app data and protects server-side credentials, and Lavalink handles music search and audio metadata.

## Repository Structure

```text
wavelet/
├── lib/                    # Flutter source code
├── android/ ios/ web/      # Flutter platform targets
├── backend/                # Node.js API
├── lavalink/               # Lavalink configuration and docs
├── docs/                   # Project documentation
├── assets/                 # App assets and branding
├── .github/                # Issue templates and CI workflows
├── README.md
├── LICENSE
├── .gitignore
├── .env
└── CONTRIBUTING.md
```

If you prefer a strict monorepo layout, the Flutter files can later be moved into `frontend/`. This repository currently keeps the Flutter app at the root, which is the standard layout generated by Flutter.

## Features

- Spotify-style Flutter interface for home, search, library, liked songs, playlists, settings, mini player, and now playing.
- Node.js Express backend for search proxying and local JSON storage.
- Lavalink integration for music search and track metadata.
- Queue, recent tracks, playlists, liked tracks, shuffle, repeat, and autoplay flows.
- Configurable backend URL for emulator, physical device, LAN, or tunnel usage.
- Secret-safe setup: real `.env` files, API keys, OAuth secrets, Discord credentials, and Lavalink passwords are ignored.

## Installation

```bash
git clone <repository-url>
cd wavelet
cp .env.example backend/.env
```

Update `backend/.env` with local values. Never commit real secrets.

## Backend Setup

```bash
cd backend
npm install
npm start
```

Default API URL:

```text
http://localhost:3000/api
```

Health check:

```bash
curl http://localhost:3000/api/health
```

## Lavalink Setup

1. Install Java 17 or newer.
2. Place `Lavalink.jar` in `lavalink/` locally, or download it during deployment.
3. Review `lavalink/application.yml`.
4. Keep the password in `lavalink/application.yml` and `backend/.env` aligned.
5. Start Lavalink:

```bash
cd lavalink
java -jar Lavalink.jar
```

See [lavalink/docs/setup.md](lavalink/docs/setup.md) for notes.

## Flutter Setup

```bash
flutter pub get
flutter analyze
flutter run
```

For Android emulator, use:

```text
http://10.0.2.2:3000
```

For a physical device, use your computer LAN IP:

```bash
flutter run --dart-define=WAVELET_API_BASE_URL=http://YOUR_COMPUTER_LAN_IP:3000
```
